=============================================================================================
                       DETAILANSICHT: DER EVA-ZYKLUS (SystemController)
=============================================================================================

[ 1. INPUT (Lesen) ]
  -> ReadAdcInputs()       : Liest I2C-ADCs -> MovingAverageFilter -> ProcessImager (Update Puffer)
  -> ProcessSerialRx()     : SerialStreamer liest UART -> Timeout/Puffer -> API Event
  -> FetchMailboxTargets() : Entnimmt asynchrone API-Ziele aus der Mailbox -> ProcessImager

         |
         | (Datenübergabe in den Process-Schritt)
         v

[ 2. PROCESS (Verarbeitung) ] - [MUTEX LOCKED]
  -> ProcessImager::Sync   : Kopiert Update-Werte in das aktive Abbild, setzt Change-Flags.
  -> RegulateAnalogOut()   : P-Regler (Vergleicht Soll-Wert mit Readback-ADC).
                             Greift bei >10% Abweichung ein (Kurzschluss-Schutz -> 0V).
  -> CheckDigitalOutSafety(): Prüft 80%-Referenzschwelle beim Schalten von SPS Signalen.
  -> EvaluateErrorState()  : Prüft globale Fehler für die System-LED.

         |
         | (Change-Flags triggern Output-Updates)
         v

[ 3. OUTPUT (Schreiben) ]
  -> UpdateDacOutputs()    : Sendet berechnete Regler-Werte aus Imager an den DAC.
                             Schaltet GPIO-Fail-Safe bei I2C-Fehler.
  -> UpdateDigitalOutputs(): Schaltet physikalische GPIOs basierend auf Imager-Status.
  -> UpdateHardwareLeds()  : LedManager berechnet Farben (Rot=Float, Grün=OK) -> I2C LEDs.
  -> UpdateSystemLed()     : LedManager fragt ErrorManager ab -> System-LED Rot/Grün.
  -> ResolveErrors()       : ErrorManager überträgt Shadow-Errors in das offizielle Register.
